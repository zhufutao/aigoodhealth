# Design

## Theme

An editorial operations console with restrained tinted neutrals, disciplined spacing, and a single confident green accent. The physical scene is a daytime desktop workflow: a content operator checks many records, opens drawers, copies text, and downloads assets while keeping health-compliance boundaries in view.

## Color

Use OKLCH tokens. Neutrals are warm, never pure black or pure white. The accent is used for primary actions, focus, selected navigation, and success states. Copper is reserved for low-frequency metadata emphasis. Red is reserved for destructive and risk states.

## Typography

Use a native product UI stack for all controls and content. Avoid display fonts in app chrome. Keep headings compact and fixed-size, with hierarchy coming from weight, spacing, and placement.

## Layout

Use a stable side navigation on desktop and a collapsed stacked layout on small screens. Pages should prioritize scanning: page header, primary form or action panel, then list/detail content. Drawers are for record inspection and editing without losing list context.

## Components

Buttons, inputs, panels, cards, drawers, modals, pagination, copy blocks, and image asset blocks should share one visual vocabulary: 8px radius, visible borders, restrained shadows, consistent focus rings, and predictable hover states.

## Motion

Motion is short and functional: hover, focus, modal/drawer presence, loading progress. Respect reduced-motion preferences.
